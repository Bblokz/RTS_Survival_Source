Radixite growth component.

You will need to implement a new cpp class for the radixite groth feature.
the radixite growth will have 2 main parts to it:
expanding with growth decals(these will be material instances specifically) and expansing with groth nodes (these can be any actor derived class)
The component needs to be configurable completely from blueprint edit defaults and will have a few edit anywhere settings.
edit anywhere:
growth direction bias: this will be a bitmask iteger that has options:
no preference,
World Pos X,
World Neg X,
World Pos Y,
World Neg Y,

when no preference is part of any of the combinations then we do not take into account any growth direction bias.

growth direction bias probabilty: between 0 and 1, this will be the probability that we will use the growth direction bias when we are calculating growth directions.
this will not be used if the growth direction bias is set to no preference.

edit defaults settings:
float IntervalNodeGrowth : how long between each growth node spawn, this will be used to determine how often we spawn growth nodes as the radixite grows.
float IntervalDecalGrowth : how long between each growth decal spawn, this will be used to determine how often we spawn growth decals as the radixite grows.

an array of FGrowthDecalOptions which contains a decal material instance and a float that is called AimAtRotationYaw,
this rotation needs to be aplied for it to look like the decal aims at the direction of the growth, it needs to work in world Z but note
that decal actors are always placed with -90,0 ° on the Y rotation so calculate properly for this aim rotation to work as intentded.
it will also have a float DistanceCoveredAtScale1 which determines how far a decal reaches per scale of it. 
so if set to 150 then when we scale the decal uniformly by 2 it will reach 300 units in the world.

float MinDecalExpansionDistance, float MaxDecalExpansionDistance; these two settings will determine the minimum and maximum distance that the growth decals can expand when they spawn, this will be used to add some variation to the growth decals as they spawn.
float DecalRotationOffset; the yaw angle used to rotate the next decal when expanding a decal from another decal; this will be used to have simmilair offset between decals in terms of their angle and it not being pure random which can
look noisy.

underneath the array of FGrowthDecalOptions we will have
int32 MaxDecalGrowth which will determine how many growth decals can be spawned at max, once this number is reached we will stop spawning growth decals until growth decals are destroyed (more on that later)

a struct, completely bp friendly called FGrowthNodeOptions:
containing an array of actor classes.
float MinRotation and float MaxRotation,
int32 MaxSpawnedNodes how many it can spawn maximially. (after max nodes are spawned stop the spawning sequence and it will wait till an actor is destroyed then it will restart the spawning sequence)
an int32 MaxDecalGrowthPerNode; for how many 'branches' of growth decals can start from a node.

we also need a min and max int for how many decal growths we will have till we spawn a growth node. a growth node always needs to be spawned at the point a growth decal was aimed at but that needs to be the last
decal of a 'branch' so lets say during the decal growth we pick to spawn 3 in a branch then the growth node potential point will be at the end of that branch.

we also need a setting int32 MaxRetries; for how many times we attempt either type of spawn, when exhausted we wait for the next iteration.

we also need a float ZLandscapeTraceOffset; that will be used to find the landscape height by tracing in 1/2 this above and 1/2 this below the point we want to spawn at, this will be used to make sure we spawn at the correct height on the landscape and not in the air or underground.

an FVector ProjectionToNavMeshExtent; to use to find locations.

how the spawning needs to work:
for decal growth:
pick one of our growth node points to start from that is not yet saturated (has not reached MaxDecalGrowthPerNode) then pick a direction using our bias if the bias probability does not win
or we do not have a bias then we pick a random direction with yaw in all multiplicatives of DecalRotationOffset so if that is 10 then we pick in {0, 10, 20 ,... 350}
we pick random in the range of MinDecalExpansionDistance and MaxDecalExpansionDistance for how far we want to expand in that direction, then we
do a landscape trace in that direction from our start point with distance we use our ZLandscapeTraceOffset for the lenght of that trace which is completely in world Z.
if we hit a location then project hte location to the nav mesh with our projection extent (ensure the navigation system is cached at begin play!!)
if that succeeeds then:
 we check with FRTSPathFindingHelpers::IsLocationInHighCostArea whether this point is at a High cost nav area
if it is then this point is not valid (we need low cost) try again at a different point (till exhaustion)
if the point failed to project or hit earlier also try again.
if the point is cost valid then place the decal. after that start from that decal point (if there are still decals left for this 'branch')
and use either - or + the offset for the angle, use the same distance (and hence the same scaling) and try placing idem.
do this till the branch is complete.

IMPORTANT NOTE: do keep track of with which offset the (first of a branch) decals from a growth node point are spawend so that we never spawn over another decal with the same offset from that growth point!

for placing the growth nodes: at a growth position that is vacant. note that new growth positions are already validated by placing those decals so make sure those projected and validated positions are cached.
